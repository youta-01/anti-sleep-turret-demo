# ADR 005: 強制より安全を優先する

## Context

Astは通常の回避に抵抗するよう設計され、その結果、停止のコストも高くなりました。その後、アプリケーションの不具合により一般のPC利用を妨げる危険が生じました。アプリ内Emergencyボタンは、アプリ自身が正常に応答することへ依存するため十分ではありませんでした。

## Decision

安全を強制より上位に置き、State MachineとアプリUIの外側にTechnical Safety Cutoffを設けます。公開版では設計境界だけを示し、具体的な停止コマンドや非公開経路は公開しません。

## Alternatives

- アプリ内Emergencyボタンだけに依存する
- 強制機能をすべて撤去する
- 不具合ごとに個別復旧手順を追加する

## Trade-offs

外部停止は障害分離を改善しますが、別経路の保守、誤作動防止、実Windowsでの受け入れ試験が必要です。また、回避耐性との緊張関係を完全には解消しません。

## Observed consequence

失敗をUI上の例外ではなく、アプリから独立した安全境界の問題として扱う設計へ変わりました。

## Current status

採用方針です。ただし現在のTechnical Safety Cutoffが完全に検証済みとは主張しません。実Windowsでの受け入れ強化は継続中です。
