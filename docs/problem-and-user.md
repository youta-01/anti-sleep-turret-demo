# 問題と対象ユーザー

## 対象ユーザー

Astの対象ユーザーは作者本人です。約4か月のCLL（作者の自己管理ログ）を振り返る中で、日中の寝落ちがCG制作に使える時間を失わせる主要なボトルネックとして見つかりました。目的は睡眠を診断することではなく、作者がCG制作へ使える時間を守ることです。

## なぜ入浴なのか

通常のアラームは、止めた後も行動が変わらないことがあり、作者のケースでは十分な介入になりませんでした。そこで、検知後の中心的な介入として入浴を選び、警告から入浴開始・完了までを状態遷移にしました。

一方、回避しにくさだけを強めると、通話・試験・休憩・障害対応など、正当な利用を妨げます。Astは介入の強制力と継続利用可能性を両立させるため、段階的警告、明示的な例外、Presence Guard、安全上の外部停止境界を組み合わせています。

## 対象外

Astは次を目的にしていません。

- 運転中の監視
- 産業用途の監視
- 医療上の診断
- 治療
- 他者の行動制御

これは作者本人のN=1プロトタイプであり、一般利用者への有効性は確認していません。
